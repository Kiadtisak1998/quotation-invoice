รายการรออนุมัติ
แสดงเฉพาะเอกสารที่ถึงคิวของ Role คุณ — อนุมัติ/ปฏิเสธได้จากแถวโดยตรง หรือกด "เปิด" เพื่อดูเอกสารเต็มก่อนตัดสินใจ
จัดการผู้ใช้งานและสิทธิ์ (Admin)
หน้านี้ใช้สำหรับกำหนด/เปลี่ยน Role ของผู้ใช้ที่สมัครสมาชิกแล้วเท่านั้น — การ "เพิ่ม" ผู้ใช้ใหม่จริง ๆ ต้องให้คนนั้นสมัครสมาชิก (หน้า Login → สมัครสมาชิก) ด้วยตัวเองก่อน 1 ครั้ง (การสร้างบัญชี login ให้คนอื่นโดยตรงจากหน้านี้ทำไม่ได้ เพราะต้องใช้ Service Role Key ของ Supabase ซึ่งไม่ปลอดภัยที่จะฝังไว้ในไฟล์ที่รันบนเครื่อง user) หลังสมัครแล้วจะมาปรากฏในตารางนี้พร้อม role เริ่มต้น "รออนุมัติ"
ชื่อที่แสดงUsernameRole ปัจจุบันเปลี่ยน Roleสมัครเมื่อ
ยังไม่มีผู้ใช้งาน หรือยังไม่ได้เชื่อมต่อ Supabase

ค่านี้เป็นค่าเริ่มต้นของทั้งบริษัท เอกสารใหม่จะดึงชื่อ/ลายเซ็นนี้มาใส่ให้อัตโนมัติ — หากลูกค้ารายใดต้องใช้ผู้เซ็นคนละคน สามารถตั้งค่าทับเฉพาะรายที่เมนู Customers ได้
ค่าเริ่มต้นการคำนวณ
ตัวอย่าง: {PREFIX}{YYYYMM}-{SEQ3} → QT202607-001
เลขจะเริ่มนับใหม่จาก 001 ของเดือนปัจจุบัน (เดือนถัดไปรีเซ็ตให้อัตโนมัติอยู่แล้ว เพราะ Running Number แยกตามเดือน)
สำรองข้อมูล / Backup & Restore
Backup ครอบคลุม: เอกสารทั้งหมด, ลูกค้า, สินค้า/บริการ, ข้อมูลบริษัท, บัญชีธนาคาร, Settings
Cloud Sync — เชื่อมต่อฐานข้อมูล Supabase
ไม่บังคับ — หากเว้นว่างไว้ ระบบจะใช้ LocalStorage ในเครื่องเหมือนเดิมทั้งหมด เมื่อกรอกและเชื่อมต่อสำเร็จ ข้อมูลลูกค้า/เอกสาร/รายการสินค้าจะถูกซิงค์ขึ้น Supabase โดยอัตโนมัติทุกครั้งที่บันทึก และ LocalStorage จะยังคงทำหน้าที่เป็นระบบสำรองเสมอ หากเชื่อมต่อ Supabase ไม่ได้ระบบจะกลับมาใช้ข้อมูลในเครื่องแทน
ยังไม่เชื่อมต่อ
ต้องสร้างตาราง customers / documents / document_items / company_settings / activity_logs ในโปรเจกต์ Supabase ก่อน (ดูไฟล์ supabase-schema.sql ที่แนบมาด้วย) — คัดลอกไปวางใน Supabase SQL Editor แล้วกด Run ครั้งเดียว
การเชื่อมต่อภายนอกอื่น ๆ (เตรียมรองรับอนาคต)
